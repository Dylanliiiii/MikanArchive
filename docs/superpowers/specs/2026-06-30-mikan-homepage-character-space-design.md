# MikanArchive 主页角色空间设计规格

日期：2026-06-30

## 目标

把主页从普通内容卡片首页调整为“轻二次元角色空间 + 真实知识收藏入口”的正式首页。页面第一屏负责建立头像、角色线稿和站点记忆点；下滑后的内容区负责展示真实文章、收藏、站点概况、整理流和浏览信号。

## 已确认视觉方向

- 顶部导航继续使用当前真实导航组件，保持可点击、下拉、搜索和主题切换能力。
- 首屏保留圆形头像，导航左侧 logo 使用同一头像的小尺寸版本。
- 首屏右侧大人物只使用 `anime_lineart_transparent.png` 生成的透明线稿资产，不再使用带白底的头像图。
- 首屏只保留一个右上角轻卡片，不遮挡人物线稿，不超过导航横幅的水平范围。
- 首屏底部不展示详细内容，只保留向下探索提示。
- 第二屏开始展示跑马灯、最近文章、精选收藏、个人介绍、友链一角、站点数据和整理流。
- “站点数据”展示建站天数、最近更新、文章数量和收藏数量；访问量只放在下方“浏览信号”胶囊中。
- 访问量不能伪造；未接入可读统计来源时显示“统计未接入”。

## 数据来源

- 最近文章来自 `getSortedPostsList()`。
- 精选收藏来自 `getMikanResources()`，优先 `featured`，不足时用普通收藏补位。
- 友链预览来自 `getMikanFriends()`。
- 更新记录来自 `getMikanUpdates()`，不足时退化使用最近文章时间。
- 建站天数来自 `siteConfig.siteStartDate`。
- 最近更新优先使用文章 `updated` / `published` 和 `records/updates` 中的最近日期。

## 动效要求

- 首屏进入时：导航、头像、状态徽章、标题、描述、操作入口和右侧线稿分层入场。
- 标题不只做普通淡入；使用遮罩扫入或分行揭示，形成角色海报感。
- 右侧线稿有低强度漂移和随滚动位移，不能抢正文阅读。
- 第二屏内容使用滚动触发的遮罩显现、线条展开、标签轨道漂移和胶囊 hover，而不是简单从下方浮出。
- 动效需支持 `prefers-reduced-motion: reduce`，减少或关闭大位移动画。
- 移动端保留首屏核心信息，隐藏右上轻卡片，内容单列排列，不能横向溢出。

## 导航与交互

- 顶部真实导航必须继续可点击。
- 首页操作入口分别链接到文章、工具导航和个人介绍。
- 资源标签、文章条目和友链预览都应使用真实链接。
- 状态徽章保持头像右下角，hover 后展开状态文字。
- 浏览信号胶囊 hover 后切换为统计状态；未接入统计时显示不可用说明。

## 验收标准

- `/` 首页可访问，顶部导航真实可点击。
- 主页使用透明线稿大背景，不再出现白底方块或方形边界。
- 首屏不开列出第二屏内容。
- 下滑后内容区不重复展示同一类信息。
- 站点数据与访问量分工清晰。
- 桌面和移动端无明显文字重叠、内容遮挡或横向溢出。
- 通过内容同步、页面结构测试、Astro 类型检查和构建验证。

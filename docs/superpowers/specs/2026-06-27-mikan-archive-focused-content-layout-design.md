# MikanArchive 非主页单列布局与工具导航设计规格

日期：2026-06-27

状态：已实现，最终实现与本规格一致；视觉验证覆盖 1440×900 桌面视口与 390×844 移动视口。

## 背景

现有 Firefly 重构把首页使用的壁纸横幅、人物资料、公告、分类、标签、站点统计和日历侧栏同时带到了所有页面。对文库、收藏、联系我、个人介绍和文章阅读来说，这些重复组件占用过多首屏空间，也削弱了页面本身的内容层级。

本次设计继续保留 MikanArchive 的轻二次元背景与 Firefly 导航体验，但明确把首页和非主页拆成两种布局：

- 首页继续承担氛围展示，保留大幅壁纸、人物资料和侧栏小组件。
- 非主页承担阅读、查找和管理，改为导航栏下方直接开始的单列内容布局。

工具导航以 fqzlr/my-blog 公开仓库中的 `collections.astro` 与 `collections.css` 为主要结构参考。可以复用其 MIT 许可下的布局与交互思路，但不能复制博主的私人内容、图片、账号资料或站点文案，并继续保留 Firefly 与 fuwari 的版权声明。

## 已确认决策

- 所有非主页移除大幅页面标题横幅。
- 所有非主页移除人物资料、公告、分类、标签、站点统计、站点信息和日历等通用侧栏。
- 首页视觉与侧栏结构保持不变。
- 文章详情页保留独立悬浮目录；它属于阅读辅助，不属于通用博客侧栏。
- 文章目录只在宽屏显示，移动端继续使用折叠目录或浮动入口。
- 工具导航按照 fqzlr 开源页面的结构和细节重做，MikanArchive 可以调整颜色、字体、背景与圆角。
- MikanArchive 项目后续涉及 UI、布局、样式或交互的任务默认启用浏览器视觉对比，不再重复询问用户是否启用。

## 布局架构

### 首页布局

首页继续使用现有 `MainGridLayout.astro`：

- 保留壁纸横幅与首页标题。
- 保留人物资料、公告、分类、标签、统计、日历等侧栏组件。
- 保留当前 Firefly 风格的玻璃导航、背景过渡和首页内容网格。

### 非主页布局

新增职责单一的 `ContentGridLayout.astro`，供所有非主页使用：

- 顶部使用现有导航栏，并保持 sticky。
- 当站点配置把 sticky 导航提升为固定定位时，内容容器必须预留完整导航高度，页面内容不得在首屏被导航遮挡。
- 内容从导航栏下方直接开始，不预留横幅高度。
- 使用居中的单列容器，普通页面最大宽度约为 `64rem` 至 `72rem`。
- 保留页脚、搜索、亮暗色切换、返回顶部等通用能力；不再渲染导航下方通用分类快捷栏，避免“主页 / 归档 / 分类 / 更多”在各页面重复出现。
- 不渲染 `SideBar`、壁纸横幅标题、人物卡、公告、统计和日历。
- 保留 Swup 或现有页面过渡所需的容器 ID，避免导航回退为整页刷新。

应迁移到非主页布局的路由包括：

```text
/posts/
/posts/[...slug]/
/archive/
/categories/
/resources/tools/
/resources/clips/
/friends/
/guestbook/
/about/
/search/
/rss/
/404/
```

### 文章详情目录

文章正文保持单列阅读卡片，在宽屏右侧单独悬浮目录：

- 目录不参与普通页面网格，也不重新引入博客通用侧栏。
- 目录随正文标题同步高亮。
- 中等与移动视口隐藏悬浮目录，继续使用现有浮动目录入口。
- 正文宽度、代码块、图片和上一篇 / 下一篇不得被目录挤压到难以阅读。

## 工具导航页面

### 页面头部

工具导航从导航栏下方直接开始：

- 小号英文眉题 `Collections`。
- 主标题 `工具导航`。
- 一行简短说明。
- 不使用独立的大白色标题卡，不显示返回收藏总览的大块占位区域。

### 分类筛选

分类栏参考 fqzlr 的胶囊结构：

- 整体使用一个有边框的可换行容器。
- 包含“全部”和所有分类。
- 每个分类后显示数量徽章。
- 当前分类使用滑动填充指示块。
- 点击分类只显示该分类卡片，并把选择写入 URL hash，刷新后仍能恢复。
- “全部”模式按分类分组显示。

### 分类分组

每个分类由以下内容组成：

- 标签轮廓小图标。
- 分类名称。
- 当前分类数量。
- 延伸到容器右侧的细分隔线。

### 工具卡片

桌面端三列、中等视口两列、移动端一列。每张卡片包含：

- 与工具对应的 Logo 或 Iconify 图标。
- 工具名称。
- 两行以内的用途说明。
- 底部域名。
- hover 时出现的右上角外链箭头。
- 轻微上浮、边框和阴影反馈。

卡片结构和交互密度贴近 fqzlr 开源页面，但视觉使用 MikanArchive 的薄荷绿、浅蓝、暖黄、柔粉和中性色。页面背景保持轻柔渐变或低对比装饰，不改成纯白空页。

## 数据模型

`MikanResource` 为工具导航增加可选字段：

```ts
icon?: string;
```

`icon` 优先使用项目已安装的 Iconify 图标名称，例如：

```text
simple-icons:openai
simple-icons:claude
simple-icons:astro
simple-icons:cloudflare
```

无图标时使用通用工具图标兜底。公开示例数据需要覆盖至少三个分类，并包含足够卡片验证三列、两列和单列布局；示例不能冒充真实私人收藏。

## 其他收藏页面

- 收藏一级导航只作为下拉分组，不再设置收藏总览功能页；旧 `/resources/` 路径保留为到 `/resources/tools/` 的兼容跳转。
- 工具导航和摘录收藏均改用非主页单列布局，并在正文区域使用白色 / 半透明 section 方框作为内容打底。
- 摘录收藏的标题字号、英文眉标、图标和描述文案节奏与工具导航保持一致。
- 摘录收藏的分类标签复用工具导航的胶囊风格；摘录分组复用工具导航的分类图标、数量和分割线结构。
- 摘录条目改为整张卡片可点击跳转原文，卡片使用略带暖色的底色、轻边框、底部 / 右侧层级阴影，并在 hover 时出现彩色阴影反馈。
- 工具导航和摘录收藏继续共享 `kind` 数据模型与分类 helper。

## 文章总列表

- `/posts/` 页面继续使用非主页单列布局和工具导航同款标题系统。
- 文章按分类标签分组，并通过顶部分类标签胶囊筛选。
- 分组内每篇文章使用无封面长条卡片；即使文章 frontmatter 中存在 `image` 字段或正文中包含图片，也不在总列表展示封面。
- 每条长条卡片整体作为进入文章详情的链接，避免只能点击标题、封面或右侧按钮。
- 长条卡片展示标题、发布日期、更新时间、兼容分类、摘要和标签；移动端保持单列，不产生横向溢出。

## 响应式与可访问性

- 桌面端工具卡片三列，平板两列，移动端一列。
- 分类胶囊允许换行，不能造成横向滚动。
- 页面在 `390px` 视口不得出现横向溢出。
- 筛选按钮使用真实 `button`，保留键盘焦点样式与 `aria-pressed` 状态。
- 外链卡片保留 `target="_blank"` 与 `rel="noopener noreferrer"`。
- 文章目录在移动端隐藏时必须仍有可访问的目录入口。

## 测试与验证

自动测试至少覆盖：

- 非主页使用单列内容布局，首页仍使用原首页布局。
- 非主页布局不渲染通用双侧栏和横幅标题。
- 工具数据支持 `icon`，无图标时有兜底。
- 分类数量与实际工具数量一致。
- 分类按钮与内容分区一一对应。
- 工具卡片包含图标、标题、说明、域名和外链。

完整验证命令：

```bash
npm run sync:content
npm run validate:content
npm run test:content-model
npm run test:pages
npm run check
npm run build
```

浏览器验证：

- 首页确认横幅和侧栏仍存在。
- 工具导航确认无横幅、无通用侧栏、分类筛选可用。
- 工具导航、摘录收藏、文章、分类标签、友链、留言、个人介绍页确认均为单列；所有非主页不显示顶部通用分类快捷栏。
- 摘录收藏确认整卡点击可跳转原文，hover 时卡片边框、阴影和提示按钮有反馈。
- 文章详情桌面端确认悬浮目录可用，移动端确认正文无溢出。
- 桌面与 `390px` 移动视口均检查控制台错误和横向溢出。

## 验收标准

- 首页视觉不退化，仍保留现有横幅和侧栏。
- 所有非主页首屏不再出现大幅标题壁纸。
- 所有非主页不再出现人物、公告、统计和日历通用侧栏。
- 所有页面不再显示导航下方通用分类快捷栏。
- 工具导航在结构、密度和交互细节上接近 fqzlr 开源页面。
- 工具卡片均有对应图标或明确兜底图标。
- 文章详情宽屏保留独立目录，移动端阅读正常。
- 亮暗色模式、GitHub Pages base path 和内容仓库分离继续可用。

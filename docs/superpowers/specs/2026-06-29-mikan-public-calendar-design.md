# MikanArchive 公开日历设计规格

日期：2026-06-29

## 背景

用户希望在 MikanArchive 的“我的”区域新增一个日历功能。最初考虑过连接 Google Calendar，但经过隐私评估后决定不接入 Google，不展示私人日程，也不在公开站点实现前台创建或编辑日程。

本规格定义的是 MikanArchive 自带的公开日历页面。它用于展示公开节日、农历、纪念日、学习/项目安排、站点维护节点和文章发布日。日程由内容文件维护，后续用户可以把需要新增的公开日程交给维护者写入内容数据。

## 参考实现结论

已阅读 fqzlr/my-blog 的公开日历实现。该实现具备：

- `calendarConfig.ts` 配置节日 API、内置节日、生日、纪念日和自定义安排。
- `holidays.json.ts` 在构建期拉取 `timor.tech` 中国法定节假日 API，并合并内置农历节日。
- `lunar-typescript` 用于公历与农历换算。
- `calendar-events.ts` 聚合 holiday、birthday、schedule 和 post 四类事件。
- `/calendar/` 页面通过 Svelte 组件展示左侧最近事件、中间月历网格和右侧当天事件详情。

该参考实现不包含：

- Google Calendar 同步。
- 前台创建或编辑日程。
- 年视图、周视图和日视图。
- 24 小时时间轴。
- 数据库、登录或公开写入能力。

MikanArchive 可以借鉴它的节日预生成、农历换算、事件聚合和月历展示思路，但需要独立设计多视图体验和内容分离数据模型。

## 已确认决策

- 不连接 Google Calendar。
- 不做访客创建日程。
- 不做站内后台编辑日程。
- 不引入数据库、登录系统或用户注册。
- 日程由公开内容文件维护，用户后续提供公开日程后由维护者写入内容数据。
- 页面支持年、月、周、日视图切换。
- 周视图和日视图使用类似 Google Calendar 的 24 小时时间轴，只展示日程，不支持拖拽编辑。
- 页面作为公开功能页进入“我的”下拉。
- 示例内容不能冒充真实私人安排，也不能写入真实敏感日程。

## 信息架构

顶部导航仍保持：

```text
主页 / 文库 / 收藏 / 联系我 / 我的
```

“我的”下拉调整为：

```text
个人介绍 / 站点概览 / 日历
```

新增页面：

```text
/calendar/
```

职责：

- 展示公开日历。
- 承载节日、农历、纪念日、公开日程和文章发布日。
- 为用户未来维护公开日程提供清晰的数据模型和示例。

首页可以后续保留一个轻量“近期日程”或“迷你日历”入口，但第一版重点是独立 `/calendar/` 页面。

## 页面布局

采用“博客混合版”布局，不做办公软件式沉重界面。

桌面端结构：

```text
左侧概览栏 / 中间主日历 / 右侧详情栏
```

左侧概览栏：

- 今日摘要。
- 最近节日。
- 最近日程。
- 最近文章发布日。
- 类型筛选。

中间主日历：

- 顶部标题与年月导航。
- 年 / 月 / 周 / 日视图切换。
- 今天按钮。
- 当前筛选状态。

右侧详情栏：

- 当前选中日期。
- 农历信息。
- 当天事件列表。
- 事件详情、备注和文章链接。

移动端结构：

- 顶部保留紧凑标题与视图切换。
- 概览栏折叠为横向卡片或顶部摘要。
- 右侧详情移动到主日历下方。
- 周视图和日视图允许纵向滚动，不产生页面横向溢出。

## 视图设计

### 年视图

年视图用于快速了解一年中哪些月份和日期有事件。

第一版建议：

- 展示 12 个月迷你月历或月份卡片。
- 每月显示事件数量、节日数量和日程数量。
- 有事件的日期显示彩色圆点或低饱和色块。
- 点击月份进入月视图。
- 点击具体日期进入日视图或更新右侧详情。

### 月视图

月视图是默认视图。

要求：

- 使用 7 列日历网格。
- 显示公历日期与可选农历短文本。
- 展示节日、纪念日、公开日程和文章发布日的事件胶囊。
- 同一天事件过多时显示前几条和 `+N`。
- 点击日期更新详情栏。
- 点击非当前月日期时跳转到对应月份。

### 周视图

周视图用于展示“像 Google Calendar 一样”的时间安排感。

要求：

- 默认显示周一到周日；如果需要更学习/工作导向，可配置为只突出周一到周五。
- 左侧为 0:00 到 24:00 的小时刻度。
- 顶部为每一天的日期列。
- 有开始和结束时间的日程按时间段渲染为彩色块。
- 全天事件显示在顶部全天区域。
- 无时间的普通事件显示在全天区域或当天摘要中。
- 当前时间线可以作为后续增强，第一版可选。
- 仅展示，不支持拖拽、缩放或直接编辑。

### 日视图

日视图用于查看某一天的完整安排。

要求：

- 左侧或主区域显示 0:00 到 24:00 时间轴。
- 有时间范围的日程按位置和高度显示。
- 全天事件、节日、农历和文章发布日放在顶部摘要。
- 右侧或底部显示事件详情。
- 移动端以单日纵向时间轴为主。

## 数据模型

公开框架仓库保留示例数据：

```text
content.example/calendar/events.json
```

未来私有内容仓库可覆盖：

```text
content/calendar/events.json
```

推荐事件字段：

```ts
type CalendarEventKind =
  | "holiday"
  | "anniversary"
  | "schedule"
  | "site"
  | "post";

type CalendarEvent = {
  id: string;
  title: string;
  kind: CalendarEventKind;
  date?: string;              // YYYY-MM-DD，一次性全天事件
  start?: string;             // ISO 或 YYYY-MM-DDTHH:mm
  end?: string;               // ISO 或 YYYY-MM-DDTHH:mm
  allDay?: boolean;
  note?: string;
  color?: "pink" | "blue" | "mint" | "yellow" | "neutral";
  icon?: string;
  url?: string;
  recurring?: {
    freq: "weekly" | "monthly" | "yearly";
    weekday?: 0 | 1 | 2 | 3 | 4 | 5 | 6;
    month?: number;
    day?: number;
    lunar?: boolean;
  };
  visibility?: "public";
};
```

规则：

- `visibility` 第一版只允许 `public`，避免误以为支持私有日程。
- 有时间段的日程使用 `start` / `end`。
- 全天事件可以使用 `date` 或 `allDay: true`。
- 农历重复事件使用 `recurring.yearly` + `lunar: true`。
- 示例数据必须明确为公开示例，不能使用真实私人日程。

## 节日与农历

节日来源建议：

- 构建期拉取公开中国节假日 API，预生成静态 JSON。
- 内置常见农历节日作为兜底。
- API 失败时仍能通过内置节日构建。

农历能力：

- 使用成熟库进行公历/农历换算。
- 月视图显示农历日短文本。
- 事件详情显示农历月日完整文本。

注意：

- 外部节假日 API 属于构建期依赖，不能影响客户端运行。
- 如果 API 不稳定，构建应有兜底和清晰日志。

## 事件聚合

需要建立独立工具层：

- 读取公开日历事件。
- 展开一次性事件。
- 展开 weekly、monthly、yearly 重复事件。
- 合并节日、纪念日、公开日程和文章发布日。
- 按日期分桶。
- 按类型优先级排序。
- 为年/月/周/日视图提供同一份规范化事件数据。

事件类型优先级建议：

```text
holiday > anniversary > schedule > site > post
```

## 视觉方向

关键词：

```text
轻二次元收藏馆 + 清晰时间管理 + 可公开展示的个人节奏
```

要求：

- 使用 MikanArchive 现有薄荷绿、浅蓝、暖黄、柔粉和中性色，不做单一粉色。
- 月视图保持博客卡片感，避免过度办公软件化。
- 周/日时间轴要足够清晰，日程块不能遮挡文字。
- 暗色模式保持低对比背景和清晰边界。
- 图标使用现有 Iconify / Astro Icon 体系。
- 卡片和时间轴在移动端不得横向溢出。

## 不做范围

第一版明确不做：

- Google Calendar、Outlook、Apple Calendar 同步。
- ICS 导入或导出。
- 前台创建、编辑、删除日程。
- 访客提交日程。
- 登录、数据库、Cloudflare KV/D1 写入。
- 拖拽调整时间、拖拽创建日程。
- 私有日程或权限分级展示。
- 提醒通知或邮件通知。

这些能力如果未来需要，应作为第二阶段或独立规格重新设计。

## 验收标准

功能验收：

- `/calendar/` 可访问，并进入“我的”下拉。
- 无私有内容仓库时，公开示例日历可以构建和展示。
- 节日、纪念日、公开日程和文章发布日能统一显示。
- 年、月、周、日视图可切换。
- 周视图和日视图能够按时间段显示日程块。
- 点击日期或事件能更新详情或跳转文章链接。
- 无前台创建入口、无登录入口、无数据库写入。

数据验收：

- `content.example/calendar/events.json` 不包含真实私人日程。
- 内容校验能发现缺失 `id`、非法日期、非法时间范围和不支持的事件类型。
- 重复事件展开范围受控，不能生成无限事件。

视觉验收：

- 桌面端三栏或主从结构清晰。
- 移动端无横向溢出。
- 月视图、周视图和日视图均可读。
- 日程块、节日、文章发布日能用颜色和图标区分。
- 亮暗色模式均可读。

验证建议：

```bash
npm run sync:content
npm run validate:content
npm run test:content-model
npm run test:pages
npm run check
npm run build
```

浏览器验证：

- `/calendar/` 桌面 1440×900。
- `/calendar/` 移动 390×844。
- 导航“我的”下拉包含“日历”。
- 年 / 月 / 周 / 日切换无控制台错误。

## 决策

采用“博客混合版公开日历”：

- 借鉴 fqzlr/my-blog 的节日预生成、农历换算、事件聚合和月视图思路。
- MikanArchive 自行实现年/月/周/日多视图。
- 使用公开内容文件维护日程。
- 暂不实现任何站内写入能力。

# MikanArchive 站点概览与页脚状态设计规格

日期：2026-06-29

## 背景

当前“我的”下拉同时放置个人介绍、GitHub 和 RSS。用户希望把站点统计、构建信息等内容单独做成一个功能页，放在“我的”下拉里并位于个人介绍下方；RSS 和 GitHub 不继续作为“我的”下拉项，而是参考 Firefly 博客页脚样式放到站点底部。同时，站点统计中的“分类标签”和“标签”在当前信息架构下含义重复，需要保留真正有用的一项。

用户补充的公开个人链接：

- GitHub：`https://github.com/Dylanliiiii`
- B 站：`https://space.bilibili.com/37007345?spm_id_from=333.1387.0.0`

## 决策

- 新页面命名为“站点概览”，路径为 `/site/`。
- “我的”下拉顺序调整为“个人介绍 / 站点概览”。
- GitHub、B 站、RSS 移到页脚常驻链接区；RSS 不再放在“我的”下拉。
- 页脚新增“已运行 X 天 / 最后更新于 X 天前”状态。
- 站点统计只保留“文章 / 分类标签 / 总字数 / 运行时长 / 最后活动”。其中“分类标签”继续使用文章 `tags` 统计，因为项目已明确文章标签承担分类用途。
- `/site/` 使用非主页 `ContentGridLayout`，不恢复横幅和通用侧栏。

## 页面结构

站点概览页从上到下包含：

1. 标题区：`MIKAN SITE / 站点概览` 与简短说明。
2. 运行统计：文章数、分类标签数、总字数、运行时长、最后活动。
3. 站点信息：站点域名、文章许可、构建平台、博客版本。
4. 构建信息：Firefly、Astro、Node、pnpm、构建时间、系统信息。
5. 相关入口：GitHub、B 站、RSS。

## 验收标准

- `/site/` 可访问，且使用聚焦内容布局。
- “我的”下拉只包含“个人介绍”和“站点概览”。
- 页脚显示 GitHub、B 站、RSS、已运行天数和最后更新时间。
- 站点统计不再同时显示“分类标签”和“标签”；保留“分类标签”。
- GitHub 与 B 站链接使用用户提供的公开地址。
- 自动测试、类型检查、构建和桌面/移动端浏览器视觉检查通过。

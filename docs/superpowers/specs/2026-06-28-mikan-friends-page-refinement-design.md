# MikanArchive 友链页精简与筛选胶囊修复设计规格

日期：2026-06-28

状态：友链页精简已实现并验证；申请说明入口调整已确认，待实施

## 背景

顶部一级导航“联系我”已经通过下拉菜单承载“友链 / 留言 / QQ群”三个入口。用户进入 `/friends/` 后，页面不应再次展示“联系我”总览和同一组三张入口卡片；该路由只负责友链浏览、筛选和申请说明。

当前友链标签筛选复用了工具导航的白色激活文字样式，但没有渲染或定位对应的彩色滑动胶囊，导致当前标签在浅色背景上不可读。

## 已确认决策

- `/friends/` 改为纯友链页面，不再展示“联系我”标题和友链、留言、QQ群入口卡片。
- 页面内容顺序为：友链标题与说明、搜索框、标签筛选、友链卡片、无结果提示、友链申请说明、可选评论区。
- 页面主标题使用“友链”，不再在内容区重复渲染第二个“友链”分组标题。
- 标签筛选复用工具导航现有的胶囊容器、彩色滑动指示器、激活文字和数量徽章样式。
- 激活胶囊在首次载入、标签点击和视口尺寸变化后重新定位；筛选行为和无结果提示保持现有功能。
- 桌面端和移动端均不得出现横向溢出，亮色与暗色模式下激活标签都必须可读。

## 方案比较

### 方案 A：友链页内局部修复（采用）

删除联系入口区，在现有友链筛选中补齐胶囊指示器和定位逻辑。改动集中、风险最低，也能直接复用已经验证过的工具导航视觉语言。

### 方案 B：只给激活按钮添加固定背景

实现更短，但缺少工具导航同款滑动反馈，也会形成第二套激活态规则，不采用。

### 方案 C：抽取跨页面筛选组件

长期复用更完整，但本次只有友链页存在缺失，抽象会扩大测试和迁移范围，不在本次修复中处理。

## 实现范围

主要调整：

- `src/pages/friends.astro`
- `tests/mikan-pages.test.ts`
- `development-log.md`
- `docs/next-tasks.md`

如共享样式能够直接满足需求，则不修改 `src/styles/resources.css`；只有浏览器验证发现友链页存在独有的响应式或暗色问题时，才增加最小范围样式。

本次不修改导航结构、友链数据模型、示例内容、留言页和 QQ 群配置。

- 友链筛选包含工具导航同款滑动胶囊指示器。
- 页面脚本在初始化、点击和视口变化时更新胶囊位置。

## 验收标准

- 从“联系我”下拉进入友链后，页面只呈现友链相关内容。
- 页面中不再出现“联系我”大标题和三张联系入口卡片。
- 默认“全部”和点击后的任意标签都有可见的彩色胶囊底。
- 激活文字、数量徽章在亮色与暗色模式下均清晰可读。
- 友链搜索、标签筛选、卡片外链、无结果提示和申请说明保持可用。
- 自动测试、类型检查、构建和桌面 / 移动浏览器验证通过。

## 后续调整：友链申请说明弹窗

### 背景

友链申请说明属于 `/friends/` 的页面职责，不应在 `/guestbook/` 再展示“友链申请”说明卡和“查看友链说明”链接。参考页面把“如何申请友链”入口放在友链标题区右侧，用户希望 MikanArchive 先保留同类入口和文档式窗口，后续再决定是否改为独立文档路由或外部链接。

### 方案比较

- 采用：友链页原生弹窗。入口位置和参考页面一致，可以直接复用现有友链 Markdown，当前即可使用，也为未来替换目标保留清晰边界。
- 不采用：新增独立申请文档页。当前只需要先留出窗口，新路由会扩大导航、文档和内容同步范围。
- 不采用：直接跳转外部链接。当前没有确定的正式目标地址，公开仓库也不应写入虚构私人联系方式。

### 已确认设计

- 友链页标题区改为桌面端左右布局：左侧保留眉标、主标题和说明，右侧放置“如何申请友链”按钮。
- 移动端按钮换到标题说明下方，不遮挡标题，也不造成横向溢出。
- 点击按钮打开原生 `<dialog>`，窗口采用文档式布局，标题为“如何申请友链”，包含关闭按钮和现有 `friends.mdx` 内容。
- 当前不创建独立路由或外链；按钮与弹窗使用稳定的 `data-*` 契约，未来可以替换为内部文档链接或外部申请地址。
- 弹窗支持关闭按钮、点击遮罩关闭和 Escape 关闭；使用 `aria-haspopup="dialog"`、`aria-controls`、`aria-labelledby` 与 `aria-modal="true"`。
- 弹窗正文限制最大高度并允许内部滚动，亮色、暗色和移动端均需可读。
- 友链页不再把申请 Markdown 以内联大卡片放在友链列表底部，同一份内容只在弹窗中出现。
- 留言页描述不再提及友链申请；移除“友链申请”说明卡和未启用留言系统区域的“查看友链说明”链接，仅保留留言与隐私提醒。

### 验证补充

- 页面契约测试覆盖友链页按钮、原生弹窗、既有 Markdown 内容复用、打开/关闭逻辑与留言页入口移除。
- 浏览器验证覆盖桌面端标题区按钮位置、弹窗打开/关闭、遮罩与 Escape、移动端滚动、亮暗模式、焦点与页面横向溢出。

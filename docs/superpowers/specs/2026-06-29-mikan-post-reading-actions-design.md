# MikanArchive 文章页目录、浏览量与分享增强设计规格

日期：2026-06-29

## 背景

文章详情页已经使用聚焦单列布局，并在宽屏右侧保留独立文章目录。用户希望目录更接近 Firefly / fqzlr.com 的阅读辅助体验：右侧目录中能明显展示当前文章包含的标题，并有一个随滚动动态移动的小框，提示当前阅读范围。同时，文章标题元信息区需要增加浏览量和分享入口。

## 设计决策

- 右侧目录继续只在宽屏显示；中小屏继续使用现有浮动目录按钮。
- 目录内容由正文标题实时生成，不在页面中维护第二份标题数据。
- 当前阅读范围使用目录内的活动高亮框展示，并随 `IntersectionObserver` 和滚动状态更新。
- 活动项会自动滚入目录可视区，避免长文章中当前标题高亮被滚出目录面板。
- 文章标题元信息区显示 `浏览量` 与 `分享`。
- 浏览量不伪造静态计数，只复用已配置的 Twikoo / Waline / Artalk 访问统计；未配置统计服务时显示 `统计不可用`。
- 分享优先调用浏览器原生 `navigator.share`；不可用时复制当前文章链接并显示 `已复制` 反馈。
- 原有分享海报组件作为可选的底部增强能力保留，不作为本次头部分享入口的唯一实现。

## 页面结构

文章详情页标题下方保持一行元信息：

```text
发布日期 / 分类标签 / 标签 / 浏览量 / 分享
```

右侧目录保留标题 `文章目录`，目录列表外增加内层描边小框。活动高亮框在目录内部移动，不改变目录项尺寸，不造成布局跳动。

## 验收标准

- 文章页右侧目录有内层小框和随阅读位置变化的活动高亮框。
- 文章页元信息区即使未配置评论统计服务，也显示浏览量状态。
- 分享按钮可点击；支持原生分享时打开系统分享，不支持时复制链接。
- 移动端不显示右侧目录，仍可使用浮动目录。
- 目录和元信息在亮暗色模式下可读，不影响正文阅读。

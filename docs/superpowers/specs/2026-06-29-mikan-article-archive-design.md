# MikanArchive 文章归档热力图与标签时间线设计规格

日期：2026-06-29

状态：已确认，待实施

## 背景

现有 `/archive/` 已支持按查询参数过滤标签或分类，但页面没有可见的筛选入口，时间线只按年份分组，也没有文章活动热力图。此次调整参考 `fqzlr/my-blog` 的公开归档结构与 `https://tblog.mmzhiku.xyz/archive/` 的公开交互体验，在 MikanArchive 既有非主页单列布局中补齐文章热力图、标签筛选和连续时间线。

本次只借鉴公开布局、信息层级和交互方式，不复制参考站的文章、统计数据、账号资料或私人素材。Firefly 与 fuwari 的既有 MIT 版权声明继续保留。

## 已确认决策

- `/archive/` 继续作为文库中的文章归档页，不与 `/records/` 的站点维护足迹混合。
- 上方只展示博客文章活动热力图，不接入 GitHub 贡献热力图。
- 热力图的活动日期使用 `updated ?? published`；没有更新日期时回退到发布时间。
- 标签筛选采用单选模式，提供“全部”和每个标签；每项显示对应文章数。
- 标签变化后，热力图、时间线和所有统计数量同步更新。
- 当前标签写入 `?tag=` 查询参数，刷新、前进后退和分享链接都能恢复筛选结果。
- 时间线按“年份 → 月份 → 文章”组织，并使用一条连续的层级连线。
- 汇总区显示当前结果的“文章数 · 覆盖年份数”；每个年份和月份分别显示自己的文章数。

## 方案选择

采用单个 Svelte 归档组件统一管理筛选、热力图和时间线状态。

未采用的方案：

- Astro 查询参数整页刷新：实现简单，但每次选择标签都会重新加载页面，筛选反馈生硬。
- 多个独立客户端组件共享 URL：组件边界更细，但热力图、筛选和时间线之间需要额外同步协议，容易出现状态不一致。

统一组件仍会把纯数据逻辑拆到独立 TypeScript 模块，以便测试和复用；界面状态集中不等于把计算逻辑堆进组件。

## 页面结构

归档页继续使用 `ContentGridLayout.astro`，保留当前非主页单列容器和标题系统。正文从上到下为：

1. 页面标题：`MIKAN ARCHIVE / 归档` 与简短说明。
2. 标签筛选：紧凑下拉按钮，显示“归档 · 全部”或当前标签；菜单项显示标签名称与文章数。
3. 文章活动热力图：显示当前筛选结果在选定年份内的月/周分布。
4. 结果摘要：左侧显示当前范围，右侧显示“文章数 · 覆盖年份数”。
5. 连续时间线：年份、月份、文章三层结构及各层数量。

页面不恢复横幅、通用双侧栏、GitHub 热力图或参考站的其他浮动组件。

## 标签筛选

- “全部”显示所有可公开访问的非草稿文章。
- 标签列表从文章 frontmatter 动态汇总，按文章数量降序、名称升序作为同数量时的稳定排序。
- 每次只选择一个标签；再次选择“全部”清除 `tag` 查询参数。
- 标签匹配沿用精确值，不擅自改变大小写或合并近似标签。
- 不存在的 `tag` 值显示空状态，并保留“查看全部”恢复入口。
- 筛选按钮使用真实 `button`，维护 `aria-expanded`；菜单项可用键盘聚焦，当前项使用 `aria-current` 或等价状态。

## 文章活动热力图

- 每个文章只贡献一个活动日期：优先 `updated`，否则使用 `published`。
- 热力图按年份切换，每年使用 12 个月 × 4 个时间段的网格；第 1 至 7 日、第 8 至 14 日、第 15 至 21 日、第 22 日至月末分别对应 W1 至 W4。
- 色阶根据当前年份、当前筛选结果中的最高格数量归一化为 0 至 4 级。
- 每格提供中文说明，例如“5 月第 2 周：3 篇文章”，并在底部显示“少 → 多”图例。
- 只有一个年份时禁用前后切换；筛选后当前年份不再存在时，自动回到筛选结果中的最新年份。
- 空结果时显示明确说明，不绘制误导性的活动数据。

## 连续时间线与统计

- 文章按发布时间倒序排列，先按年份、再按月份分组。
- 汇总统计显示当前筛选结果的文章总数与覆盖年份数。
- 年份标题显示“共 N 篇文章”；月份标题显示“N 篇文章”。
- 文章行显示 `MM-DD`、主要标签和文章标题；标签使用 MikanArchive 的多色但低饱和视觉语言。
- 静态层级线由 CSS 绘制，悬停或键盘聚焦文章时，用一条连续 SVG 路径高亮从年份节点经过月份节点到文章节点的完整路径。
- 高亮只增强当前路径，不改变链接位置，不影响点击和键盘访问。

## 数据与组件边界

计划新增纯数据模块，负责：

- 汇总标签及数量。
- 根据单一标签过滤文章。
- 按年份与月份构建时间线分组和数量。
- 按 `updated ?? published` 构建年度热力图网格与色阶。

`archive.astro` 只负责读取文章并把可序列化数据传给归档组件。Svelte 组件负责 URL 状态、下拉菜单、年份切换、可见内容和高亮路径。生产代码不读取私有内容仓库以外的新数据源，也不改变现有文章 frontmatter 模型。

## 响应式与视觉

- 桌面端热力图完整显示月份与 W1–W4 标签，时间线显示三层连续连线。
- 移动端允许热力图区域横向滚动或按可用宽度缩小格子，但页面本身不得横向溢出。
- 移动端缩短层级缩进，保留年份、月份和文章节点关系，不隐藏统计数量。
- 文章标题超长时省略，日期和主要标签保持可辨认。
- 亮色与暗色模式都使用现有 CSS 变量和柔和的蓝、薄荷、暖黄、粉色层次，不复制参考页的纯黑配色。

## 测试与验证

测试驱动顺序：

1. 先为标签汇总、单标签过滤、年月分组、统计数量和活动日期回退编写失败测试。
2. 实现最小纯数据逻辑使测试通过。
3. 为页面结构增加契约测试，覆盖筛选入口、热力图、汇总统计和连续时间线。
4. 实现 Svelte 交互与样式，再运行完整验证。

完整验证命令：

```powershell
npm.cmd run sync:content
npm.cmd run validate:content
npm.cmd run test:content-model
npm.cmd run test:pages
npm.cmd run check
npm.cmd run build
```

浏览器验收覆盖：

- 桌面端 `/archive/` 默认全部文章。
- 选择一个标签后，URL、热力图、总数、年份数、年份数量、月份数量和文章列表同步变化。
- 刷新筛选 URL 后状态恢复。
- 年份切换、文章 hover/focus 连续路径和文章跳转可用。
- 不存在标签的空状态可恢复。
- 390×844 移动视口无页面级横向溢出，筛选菜单和热力图可用。
- 亮暗色模式下文本、色阶、节点和连线清晰，控制台无新增错误。

## 文档同步

实施时同步更新：

- `development-log.md`
- `docs/next-tasks.md`
- `CHANGELOG.md`（归档页公开体验发生明显变化）
- 与归档页面职责相关的 README 或既有正式设计说明（仅在现有描述会过期时更新）

本次不改变导航、部署命令、环境变量、内容仓库结构或项目专属 Skill 流程，因此默认不修改 `AGENTS.md`、`.env.example` 与项目专属 Skill。

## 验收标准

- 归档页顶部显示基于文章活动日期的热力图。
- 标签筛选可见、可访问、可通过 URL 恢复，并同步作用于热力图和时间线。
- 当前结果、年份和月份三级文章数量准确。
- 时间线按年、月、文章排列，并在 hover/focus 时呈现连续高亮路径。
- 页面保持 MikanArchive 的单列内容布局、配色与内容分离规则。
- 桌面与移动端通过自动测试、类型检查、构建和浏览器视觉验收。
